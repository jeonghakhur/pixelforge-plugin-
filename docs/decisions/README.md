# Architecture Decision Records (ADR)

이 디렉토리는 PixelForge Plugin 개발 과정에서 내린 주요 기술 결정을 기록한다.

## 형식

```markdown
## ADR-NNN: 제목

- 상태: 채택 / 폐기 / 대체
- 일자: YYYY-MM-DD
- 맥락: 왜 결정이 필요했는가
- 결정: 무엇을 선택했는가
- 결과: 어떤 영향이 있었는가
```

---

## ADR-001: 토스 스타일 다크 테마 채택

- **상태:** 채택
- **일자:** 2025-03
- **맥락:** Figma 플러그인 UI의 전체 테마를 결정해야 했다. Figma 에디터 자체가 다크 UI이므로 시각적 일관성이 중요했다. 밝은 테마는 Figma 에디터와 이질감을 주고, 장시간 사용 시 눈의 피로를 가중한다.
- **결정:** 토스/카카오뱅크 스타일의 계층적 다크 테마를 채택했다. `#1A1A1A` → `#2A2A2A` → `#333333` 3단계 배경으로 깊이감을 표현하고, 강조색은 `#3B82F6` (Blue-500) 단일 포인트로 절제했다.
- **결과:** Figma 에디터와 자연스럽게 어울리며, 정보 계층이 명확해졌다. 모든 색상을 CSS 변수로 토큰화하여 일관성을 유지한다.

## ADR-002: 이모지 → 배지/아이콘 교체

- **상태:** 채택
- **일자:** 2025-03
- **맥락:** 초기 구현에서 상태 표시에 이모지(✅, ❌, ⚠️)를 사용했다. 이모지는 OS/브라우저마다 렌더링이 달라 금융권 UI의 세련된 느낌을 저해했다. 특히 WCAG 매트릭스에서 이모지 크기가 불일치했다.
- **결정:** 이모지를 CSS 배지(`Pass`, `AA`, `Fail`)로 교체했다. 배지에 시맨틱 컬러(`--success`, `--warning`, `--danger`)를 적용하여 상태를 표현한다.
- **결과:** 크로스 플랫폼에서 일관된 렌더링, 배지 크기/색상을 CSS로 정밀 제어 가능.

## ADR-003: UI를 순수 HTML/CSS/JS로 구현 (프레임워크 없음)

- **상태:** 채택
- **일자:** 2025-02
- **맥락:** Figma 플러그인 UI는 iframe으로 로드되며, 번들 크기 제한이 있다. React, Vue 등 프레임워크를 사용하면 번들 크기가 불필요하게 커지고, 프레임워크 런타임 오버헤드가 발생한다.
- **결정:** 순수 HTML + CSS + Vanilla JS로 구현한다. `ui.html`에 CSS를 인라인하고, `ui.js`를 esbuild로 번들링하여 `<!-- UI_SCRIPT_PLACEHOLDER -->`에 주입한다.
- **결과:** `dist/ui.html` 단일 파일로 배포 가능. 외부 런타임 의존성 0. 빌드 시간 < 1초.

## ADR-004: Converter 모듈 분리 (JSON→CSS 변환)

- **상태:** 채택
- **일자:** 2025-03
- **맥락:** CSS 생성 로직이 `ui.js`에 모두 포함되어 파일이 비대해졌다. Variable/Color/Text/Effect 각 토큰 타입마다 변환 로직이 다르고, 테마 분리(다크/라이트) 로직이 복잡했다.
- **결정:** `src/converters/` 디렉토리에 토큰 타입별 모듈을 분리했다. 각 모듈은 순수 함수로 JSON 입력 → CSS 문자열 출력만 담당. `:root {}` 래핑은 caller(`ui.js`의 `generateCSS`)가 담당한다.
- **결과:** 각 converter를 독립적으로 테스트/수정 가능. `ui.js`의 복잡도 감소. 새 토큰 타입 추가 시 converter 하나만 추가하면 됨.

## ADR-005: 한/영 다국어 지원 (i18n)

- **상태:** 채택
- **일자:** 2025-03
- **맥락:** 초기에는 한국어만 지원했으나, 글로벌 사용을 고려하면 영어 지원이 필요했다. 그러나 외부 i18n 라이브러리는 번들 크기 제약으로 사용 불가.
- **결정:** `ui.js` 파일 내에 `i18n` 객체로 ko/en 번역을 정의하고, `t('경로')` 함수로 접근한다. HTML에서는 `data-i18n` 속성으로 자동 적용한다. 언어 전환은 헤더의 pill 버튼으로 즉시 반영.
- **결과:** 번들 크기 증가 최소화 (순수 JS 객체). 새 언어 추가 시 `i18n` 객체에 키만 추가. 동적 텍스트는 `t()` 함수 호출로 처리.
